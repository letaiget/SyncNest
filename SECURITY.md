# Security Policy

## Supported versions

At this stage, only the latest `main` branch state is supported.

## Reporting a vulnerability

Please do not open public issues for security-sensitive reports.

Use one of the following private channels:

- GitHub private security advisory (preferred)
- Direct maintainer contact (to be listed later)

Include:

- affected component
- reproduction steps
- potential impact
- optional mitigation proposal

## Disclosure process

- We acknowledge receipt as quickly as possible.
- We triage and reproduce the issue.
- We implement and validate a fix.
- We coordinate disclosure after a patch is available.
